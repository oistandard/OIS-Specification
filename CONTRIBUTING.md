# Contributing to OIS

Thank you for your interest in the Open In-Store Standard (OIS).

## Quick start

* Review the current draft in `versions/0.1.0.md`.
* Check `proposals/` for related ideas or drafts.
* Open an issue or PR with a clear description, scope, and rationale.

## Proposals

New features or significant changes should start as a proposal in `proposals/`.
Include:

* The problem statement and motivation
* Suggested changes to the spec
* Example JSON payloads or schemas
* Backwards compatibility considerations

## Style

Follow `style-guide.md` for editorial conventions.
